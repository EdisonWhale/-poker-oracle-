# AiPoker — UI/UX 设计规范

> 从 PRD v1.0 §B 抽取。桌面优先（1024px+），深色主题。

---

## 1. 信息架构

```
/                           ← 首页（双入口：创建 / 加入）
├── /room/:id?intent=create ← 等待室（创建语义）
├── /room/:id?intent=join   ← 等待室（加入语义）
└── /game/:id               ← 牌桌（实时核心）
```

### 训练提示面板（右侧可折叠）

```
┌─────────────────────┐
│ 📊 训练提示         │
├─────────────────────┤
│ 手牌强度: Top 18%   │  ← Preflop
│ 手牌强度: 权益 41%  │  ← Postflop
│ 底池赔率: 28%       │
│ 建议胜率需求: >28%  │
│                     │
│ 位置: 按钮位(BTN)   │
│ 建议: ✅ 考虑跟注   │
│ [详细说明 ▾]        │
└─────────────────────┘
```

训练 HUD 行为约束：
- Preflop 显示 `Top X%`，Postflop 显示 `权益 X%`，两者不混用。
- 只有当前用户行动时显示完整建议（suggestion / reason / pot odds / required equity）。
- 非当前用户行动时仅显示静态信息：`位置 + 手牌强度`。
- 建议来源为确定性 `TAG` 基线教练，不采样 bot 随机频率。
- `跟注`、`加注`、`弃牌`、`过牌` 文案分开显示，不再合并成“跟注/加注”。

## 3. 关键页面

**首页（/）：**
- 两个主入口卡片：`创建房间`、`加入房间`
- 创建房间：输入用户名后进入 `/room/:id?intent=create`
- 加入房间：输入用户名 + 房间号后进入 `/room/:id?intent=join`
- 房间号输入规则：自动大写、去空格、6 位长度校验、即时错误反馈

**等待室（/room/:id）：**
- 座位布局（已加入玩家 + Bot 占位）
- Bot 配置（座位位点添加，Fish/TAG/LAG）
- 同房同名冲突时阻止加入并回首页重新输入
- 进行中加入仅观战当前手，下一手参与

### 动画规范

| 动画 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 发牌 | 300ms | ease-out | 牌从牌堆飞向玩家位 |
| 翻牌 | 400ms | cubic-bezier | 3D Y轴翻转 |
| 下注 | 500ms | spring | 筹码移至底池 |
| 赢牌 | 600ms | ease-in-out | 底池飞向赢家 + 金色粒子 |
| 弃牌 | 780ms | ease-out | 两张手牌抛物线飞入弃牌堆（muck） |
| 当前行动者 | infinite | pulse | 边框脉冲光效 |

---

### Hand-End 时间线（MVP）

- `game:state` 进入 `hand_end`：展示所有在手玩家底牌（showdown 概念仍保留，但 UI 触发点统一在 `hand_end`）
- `game:hand_result` 到达后：
  - `0ms`：赢家公告入场 + 赢家座位高亮
  - `0-600ms`：底池飞向赢家 + 金色粒子爆发
  - `0-780ms`：fold/muck 补间动画（若本手有弃牌动作）
  - `2500ms`：结果状态进入 showing，可手动关闭公告
  - `4000ms`：结果状态 done，公告自动退出
  - 若 `table.canStartNextHand = true` 且当前用户是存活真人（`stack > 0 && !isBot`）：
    - `5500ms`：自动发下一手（仍可手动按钮/Space 提前触发）
  - 若 `table.canStartNextHand = true` 但当前用户已淘汰：
    - 显示“你已淘汰”双选面板（返回大厅 / 继续观战）
    - 继续观战后只显示观战条，不再显示发下一手按钮
    - 禁用该用户 `Space` 快捷键发牌
  - 若 `table.isBotsOnlyContinuation = true`（仅剩 Bot 且可继续）：
    - 服务端在 `1500ms` 后自动开始下一手
    - 客户端展示“Bot 自动继续”观战提示
  - 若 `table.canStartNextHand = false`：
    - 进入终局冻结态，底部显示“本场结束”面板与“返回大厅”主按钮
    - 禁用自动发下一手与 `Space` 手动发牌

## 5. 组件清单

| 组件 | 职责 |
|------|------|
| `PokerTable` | 椭圆形布局容器（2-9 座位自适应） |
| `Seat` | 玩家位：头像、筹码、当前下注、行动高亮 |
| `PlayingCard` | SVG 渲染，支持翻转动画、highlight |
| `CommunityCards` | 公共牌展示（Flop/Turn/River 渐次出现） |
| `HoleCards` | 底牌展示（仅自己可见；回放/showdown 展示） |
| `PotDisplay` | 主池/边池展示 |
| `ActionPanel` | fold/check/call/bet/raise + BetSlider |
| `BetSlider` | 金额输入（只允许合法区间：minBet..maxBet） |
| `ActionHistory` | 本手行动历史（可折叠） |
| `TimerBar` | 行动倒计时进度条 |
| `TrainingHUD` | 训练提示面板 |
| `ChipStack` | 筹码堆视觉（按面值分颜色） |

### 组件约束
- 渲染组件不直接依赖 socket（由 store/hook 供数据）
- 高频更新组件（TimerBar、动画）隔离状态，避免全桌重渲染
- 优先对高频更新/大组件使用 `React.memo`（其余按 profiling 决定，避免过度 memo）

## 6. 响应式

- MVP：桌面优先（1024px+）
- 断点：1440px（宽桌面）/ 1024px（普通桌面）
- 768px 以下：不在 MVP 范围，降级为"观看模式"

---

## 6.1 可见性契约（Visibility Contract）

> 该条款为牌桌交互硬约束，优先级高于视觉稿微调。

- 操作区（ActionPanel）在任何状态下（含 Raise 展开）不得遮挡以下信息：
  - 当前用户底牌
  - 当前用户倒计时（秒数+进度条）
  - 当前可执行动作的关键金额（Call / Raise / All-in）
- 牌桌页面必须采用 `Header + Table Arena + Action Dock` 三段式结构，禁止使用覆盖式 `fixed bottom` 操作栏压在牌桌内容上。
- 行动历史与训练提示应优先进入 Dock 或侧边区域，不得占用当前用户 seat 区域的视觉主路径。
- 视觉验收至少覆盖分辨率：`1366x768`、`1536x960`、`1728x1117`（常见 Mac/外接屏）。
- 验收标准：
  - 当前用户 seat 的底牌区域在视口内完整可见。
  - 倒计时最后 10 秒仍可稳定识别。
  - Raise 交互打开/关闭时不产生关键 UI 的遮挡跳变。

## 7. 快捷键

| 键 | 动作 |
|----|------|
| `F` | Fold |
| `C` | Call |
| `K` | Check |
| `R` | 聚焦 Raise 输入框 |
| `A` | All-in |
| `Space` | Check（若可）/ Call（若需） |
| `Enter` | 确认当前 Raise 金额 |
| `Esc` | 取消 Raise 输入 |
